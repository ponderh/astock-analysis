## 业务价值评估

### 一、哪些估值方法在A股最有实际投资判断价值

**第一梯队（主结论候选）：**

| 方法 | 适用场景 | A股实际价值 | 局限 |
|------|---------|-----------|------|
| **PB（市净率）** | 银行/保险/券商/周期股/资产重 | ★★★★★ | 账面价值失真时失效 |
| **股息率折现** | 高分红/公用事业/金融 | ★★★★☆ | 要求分红稳定，非成长股 |
| **DCF敏感性矩阵** | 成长股/消费/医药 | ★★★★☆ | 参数敏感，A股使用有争议 |

**第二梯队（辅助验证）：**

| 方法 | 辅助价值 | 局限 |
|------|---------|------|
| PEG | 消费/医药成长股 | A股制度断裂导致历史增速不可外推 |
| EV/EBITDA | 周期股/资本密集型 | 需标准化资本结构 |
| PS | 互联网/轻资产科技 | 收入质量参差不齐 |
| 格雷厄姆数 | 极端清零测试 | 仅适合大盘蓝筹高分红 |

**核心判断：PB应作为A股的主估值锚。** 理由：
1. A股散户占比高、市场情绪化，PB比PE更抗操纵
2. 格雷厄姆/巴菲特的价值投资实践均以PB为清算底线
3. 行业阈值库中"行业均值PB分位"是最直观的相对估值信号
4. DCF理论严谨但参数（WACC/g）主观性太强，A股机构共识难以达成

**格雷厄姆数的重新定位**：v2.2将其降级为"极端清零测试"是正确决策，但实施者可能在编码时仍会不自觉地把它当作主锚。需在代码层面强制限制其输出位置。

## 二、行业阈值库与估值集成的合理性

**集成架构评估：基本合理，但存在关键缺陷**

**合理之处：**
- 路线图中模块5（P1-1）依赖模块2/3/4数据+行业阈值库，逻辑依赖清晰
- v2.2的"行业差异化估值矩阵"（消费→PEG+DCF、银行→PB+股息率）是正确的框架设计
- "同行分位+自身历史分位"双维度比较是行业对标的标准做法

**关键缺陷：**

1. **行业分类是最大单点风险**（来自G5_review已发现的问题）
   - G5发现永新股份被分类为"医药生物"，实际为"包装印刷"
   - 错误的行业分类 → 错误的阈值 → 错误的估值判断
   - 这是系统性的、难以察觉的错误，一旦规模化后果严重

2. **阈值库与估值引擎之间缺少"数据新鲜度"校验**
   - 行业阈值有`effective_from`日期，但估值引擎调用时未必检查是否过期
   - A股行业结构变化快（3年内行业景气度可能完全改变），过期的阈值比没有阈值更危险

3. **格雷厄姆数的"无风险利率"参数未与阈值库联动**
   - v2.2提到"中国10年国债收益率重新校准"，但实现时若写死默认值则失去动态性
   - 建议：国债收益率从行业阈值库读取，或至少从配置文件读取

## 三、银行/券商/保险等特殊行业PE不适用问题

### 银行业：PB优先，PE参考价值极低

| 估值方法 | 优先级 | 原因 |
|---------|--------|------|
| PB（含拨备覆盖率调整）| ★★★★★ | 账面价值相对稳定，是清算底线 |
| 股息率折现 | ★★★★☆ | 分红稳定的银行可视为永续债替代 |
| P/贷款拨备 | ★★★☆☆ | 隐含资产质量预期 |
| PE | ⚠️ 不推荐 | 杠杆效应扭曲，盈利调节空间大 |

**实现要点：**
```python
# 行业类型识别 → 路由到不同计算函数
BANK_VALUATION_ROUTE = {
    "bank": {
        "primary": "pb",           # 上市银行PB均值比较
        "secondary": "dividend_yield",  # 股息率折现
        "avoid": ["pe", "peg"]
    }
}
```

**特殊处理：**
- 银行PB需考虑拨备覆盖率差异（覆盖率低的银行PB应打折）
- "可疑贷款/损失贷款"披露质量参差不齐时，PB比PE更稳健
- 城商行/农商行的PB估值体系与国有大行完全不同，需分组建模

### 证券业：PB优先，EV/营收为辅，PE基本无效

| 估值方法 | 优先级 | 原因 |
|---------|--------|------|
| PB | ★★★★★ | 券商净资产最稳定 |
| EV/营收 | ★★★☆☆ | 资管业务收入占比提升时有用 |
| EV/AUM | ★★★☆☆ | 财富管理转型后可考虑 |
| PE | ⚠️ 极不推荐 | 自营/资管收入波动大，一次性收益扭曲PE |

**实现要点：**
- 证券业属于强周期+政策驱动，PB历史分位比PE更有参考意义
- 熊市底部（PB<1.2）往往是更好的买入信号，而非PE低位

### 保险业：PEV优先，PB辅助

| 估值方法 | 优先级 | 原因 |
|---------|--------|------|
| P/EV（内含价值倍数）| ★★★★★ | 保险特有的估值方法，反映保单未来现金流 |
| PB | ★★★☆☆ | 投资端波动大时，EV比PE可靠 |
| PE（寿险公司）| ⚠️ 参考 | 寿险EV拆解复杂，PE作为粗略参考 |

**实现要点：**
- A股上市险企（寿险为主）需区分EV来源（个人代理人/银保渠道EV质量不同）
- EV数据每半年披露一次，时效性差，PB作为季度跟踪更实用

### 实施建议：特殊行业必须走独立分支

```python
def get_industry_category(sw_l3_code: str) -> str:
    """申万三级行业 → 估值类型路由"""
    if sw_l3_code in BANK_CODES:        return "bank"
    elif sw_l3_code in INSURANCE_CODES: return "insurance"
    elif sw_l3_code in SECURITIES_CODES:return "securities"
    elif sw_l3_code in REALESTATE_CODES:return "realestate"
    else:                               return "general"

def calculate_valuation(company, method="auto"):
    """特殊行业不走PE/PB通用路径"""
    industry_type = get_industry_category(company.sw_l3_code)
    if industry_type == "bank":
        return bank_valuation(company)      # PB优先
    elif industry_type == "securities":
        return securities_valuation(company) # PB优先
    elif industry_type == "insurance":
        return insurance_valuation(company) # P/EV优先
    # general → 标准矩阵路由
```

## 四、主结论方法的选择

**结论：PB作为A股主估值锚，DCF作为理论验证辅助**

| 场景 | 主方法 | 辅助方法 | 说明 |
|------|--------|---------|------|
| 银行 | PB | 股息率折现 | PB<0.8为深度安全边际 |
| 保险 | P/EV | PB | EV数据不可得时用PB |
| 券商 | PB | EV/营收 | PB<1.5为低估区间 |
| 消费成长 | DCF | PEG | DCF为主，PEG辅助验证 |
| 周期股 | PB | EV/EBITDA | PB历史分位比PE更有意义 |
| 科技 | PS | DCF | 成长期用PS，成熟期用DCF |
| 公用事业 | 股息率折现 | DCF | 高分红稳定股 |
| 房地产 | NAV | PB | NAV为清算底线 |

**安全边际判断的统一框架：**

```
深度安全边际：当前股价 < DCF敏感性下限 × 0.85
合理价值区间：DCF敏感性下限 × 0.85 ~ DCF中性值
高估区间：当前股价 > DCF敏感性上限 × 1.2
```

**格雷厄姆数的使用约束（必须在代码层面强制执行）：**
- 仅在`bank`/`insurance`/`high_dividend_blue_chip`行业类型中输出
- 输出时必须附带注释："注：格雷厄姆数为极端悲观清零测试，不代表正常估值锚"
- 当股价 > 格雷厄姆数时，不得将此作为任何"低估"判断的依据

## 风险与问题

| # | 严重度 | 风险 | 影响 |
|---|--------|------|------|
| 1 | 🔴 P0 | **行业分类错误导致整套估值体系失效** | 错误分类→错误阈值→错误估值。G5_review已发现永新包装被分为医药生物，此类错误难以察觉，会系统性扩散 |
| 2 | 🔴 P0 | **特殊行业PE路由缺失** | 银行/券商/保险若走通用PE计算，结果完全无意义。必须在行业识别层拦截 |
| 3 | 🔴 P0 | **格雷厄姆数被误用为主估值锚** | v2.2明确降级为清零测试，但实施者可能将其作为主要输出，框架层面的定位需要在代码中硬性约束 |
| 4 | 🟡 P1 | **阈值库数据时效性无校验** | 阈值有`effective_from`但调用端未必检查，过期阈值比无阈值更危险 |
| 5 | 🟡 P1 | **DCF参数（WACC/g）主观性过强** | A股机构对WACC共识难以达成，DCF结论可能成为"观点伪装成数字" |
| 6 | 🟡 P1 | **历史分位法的制度断裂问题** | v2.2已注明局限性（股权分置/2015泡沫/注册制），但实施时可能忘记在输出中附带免责声明 |
| 7 | 🟡 P1 | **PB分母（净资产）质量问题** | 银行关注类贷款/商誉/投资性房地产等可能虚增净资产，需设计PB调整机制 |
| 8 | 🟢 P2 | **DCF敏感性矩阵与行业阈值库未联动** | DCF的g（永续增长率）应与行业GDP增速挂钩，当前为固定范围值 |
| 9 | 🟢 P2 | **亏损公司无法使用任何盈利类估值** | 此类公司只能提供清算价值/EV/营收等备选，但报告中缺少清晰的"不适用"声明 |

## 建议优先级

### P0（上线前必须完成）

1. **特殊行业路由拦截**：在估值引擎入口层识别银行/券商/保险行业，强制路由到独立估值函数（PB/P/EV优先），通用PE计算对这些行业直接返回N/A
2. **格雷厄姆数使用约束**：在代码层面限制其输出位置和展示方式，附加强制性注释，防止被误用为正常估值锚
3. **行业分类准确性校验**：接入行业阈值库前，交叉验证SW三级分类结果（可与企查查主营构成对照），错误分类触发人工复核flag

### P1（第一版发布后快速迭代）

4. **阈值时效性检查**：get_threshold()返回时强制附带`is_stale`字段，超过N个月的阈值标记为低置信度
5. **PB分母质量调整**：对金融业PB，设计拨备覆盖率/商誉/投资性房地产等调整项
6. **历史分位法免责声明**：在输出结果中自动附注"注：历史分位法存在制度断裂局限，2015年前数据仅供参考"
7. **DCF参数范围合理性校验**：WACC超出7-15%范围时触发告警；g超出GDP名义增速时强制拦截

### P2（持续优化）

8. **DCF与行业阈值库联动**：g（永续增长率）上限与行业GDP增速中位数挂钩
9. **亏损公司估值处理**：提供清算价值下限/ EV/营收等替代指标，明确标注"PE/PB不适用"
10. **清算价值多情景**：增加"乐观/中性/悲观"三档清算价值，避免单一系数

## 特殊行业处理建议

### 银行

```
推荐估值链：PB（主）→ 股息率折现（辅）→ ROE/P90（质量验证）
禁用：PE、PEG
PB安全边际标准：
  - PB < 0.7：深度安全边际（资产质量存疑情况下可能更便宜）
  - PB 0.7-0.9：合理安全边际
  - PB 0.9-1.1：中性
  - PB > 1.3：偏高
调整项：拨备覆盖率<150%的银行PB需打折
```

### 券商

```
推荐估值链：PB（主）→ EV/营收（辅）→ 持仓自家股票/资管规模（AUM）
禁用：PE（含cyclically adjusted PE）
PB安全边际标准：
  - PB < 1.0：历史大底区间（2008/2018/2022年熊市底部）
  - PB 1.0-1.5：低估
  - PB > 2.0：亢奋区间
调整项：自营盘占比高的券商，PB应反映市场波动性折价
```

### 保险

```
推荐估值链：P/EV（主）→ PB（辅）→ 新业务价值倍数（NBV，百亿市值以上险企）
禁用：PE（寿险EV拆解复杂，PE波动极大）
P/EV安全边际标准（寿险）：
  - P/EV < 0.6：深度安全边际
  - P/EV 0.6-0.8：低估
  - P/EV 0.8-1.0：合理
  - P/EV > 1.0：溢价（反映成长性）
注：A股上市险企主要为寿险，财险可用综合成本率+承保利润率辅判
```

### 房地产（补充）

```
推荐估值链：NAV（清算底线）→ PB（相对估值）→ EV/EBITDA（压力测试）
禁用：PE（土地储备重估价值不反映在PE中）
NAV = 土地储备公允价值 - 净负债
PB > 0.5为底线（跌破净资产50%通常意味着资产折价或资不抵债预期）
```

## 总结

P1-1估值分析引擎的**业务价值高**，是整个分析系统输出"便宜/贵"判断的核心模块。但当前v2.2框架中的行业差异化矩阵是正确的，关键风险在于**实施时是否能忠实地遵循行业路由逻辑**，尤其是特殊行业的PE拦截机制。

**最核心的三个实施原则：**
1. **行业路由必须在入口处拦截**，不允许银行/券商/保险进入通用PE/PB计算分支
2. **格雷厄姆数必须降级处理**，在UI/报告中限制其展示位置，防止被误用
3. **PB是A股的主估值锚**，DCF是理论验证工具而非主结论来源

这三个原则需要在代码架构层面通过强制路由和约束性设计来保证，而非仅在文档中说明。
