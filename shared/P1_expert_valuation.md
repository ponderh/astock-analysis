# P1-1 估值分析引擎专家方案

---

## 核心方法论

### 一、A股估值方法适配性评估

| 方法 | A股适配度 | 理由 | 优先级 |
|------|----------|------|--------|
| **PE分位数法** | ⭐⭐⭐⭐⭐ | A股散户主导，情绪波动大，历史分位对判断相对高低极有价值；配合行业阈值效果更佳 | **P0** |
| **PB分位数法** | ⭐⭐⭐⭐⭐ | 周期股、券商、银行、保险、地产必备；A股破净股有特殊含义（资产质量疑虑） | **P0** |
| **格雷厄姆数** | ⭐⭐⭐⭐ | 侧重安全边际，适合价值型筛选；AShare盈利波动大，需动态EPS+PB双轨 | **P1** |
| **EV/EBITDA** | ⭐⭐⭐⭐ | 弥补PE失真（巨额非经常性损益、并购活跃）；资本密集型行业必备 | **P1** |
| **DCF** | ⭐⭐⭐ | A股贴现率(WACC)不稳定，盈利可预测性弱；适合现金流稳定的消费/公用事业龙头 | **P2** |

### 二、核心方法详解

#### P0方法1：PE/PB历史分位数法

**原理**：以个股历史估值区间为基准，判断当前处于历史什么位置。

```
PE分位 = (当前PE - 历史PE_min) / (历史PE_max - 历史PE_min) × 100%
```

**A股关键参数设定**：
- 历史窗口：**10年**（剔除2007/2015年极端泡沫干扰，可设5年+10年双窗口）
- 异常值剔除：剔除PE<0、PE>500的极值记录
- 参照基准：需按**行业(申万三级)**分别计算，A股行业估值差异极大

**行业阈值集成**（最关键）：

```sql
-- 查某股当前PE在行业内的分位
SELECT
  stock_code,
  pe_ttm,
  PERCENT_RANK() OVER (PARTITION BY sw_industry_code ORDER BY pe_ttm) AS peer_pct,
  AVG(pe_ttm) OVER (PARTITION BY sw_industry_code) AS industry_avg,
  MEDIAN(pe_ttm) OVER (PARTITION BY sw_industry_code) AS industry_median
FROM industry_valuation_snapshot
WHERE sw_industry_code = :sw3_code AND trade_date = CURRENT_DATE;
```

**输出信号**：
- 个股历史分位 < 20% + 行业分位 < 20% → 双重低估信号
- 个股历史分位 > 80% + 行业分位 > 80% → 双重高估信号

#### P0方法2：PB分位数法（银行/券商/保险专用）

**银行股特殊处理**：
```
调整PB = 市净率 / (1 + (不良率 - 行业平均不良率) × 权重)
-- 农业银行口径：不良率每高于行业均值1%，PB下调约0.05
```

**券商股**：
```
PB中枢 = f(指数Beta, 权益资产占比, 资管业务占比)
-- 权益自营比例高的券商，PB理应更低
```

#### P1方法：格雷厄姆数（Graham Number）

```
Graham_Number = sqrt(22.5 × EPS × BVPS)

A股适配修正版：
Graham_Number_Adjusted = sqrt(22.5 × EPS_ttm × max(BVPS_ttm, 1)) × 行业调整系数
```

**行业调整系数**（A股实证）：
| 行业类型 | 系数 | 原因 |
|---------|------|------|
| 消费/医药 | 1.2 | 成长性溢价，长期可持续 |
| 金融 | 0.7 | 盈利波动大，资产质量不确定 |
| 周期 | 0.5 | EPS不稳定，格雷厄姆数失效 |
| 科技 | 1.0 | 中性，成长与价值并存 |

#### P1方法：EV/EBITDA

```
EV = 市值 + 带息负债 - 现金及等价物
EBITDA = 净利润 + 所得税 + 利息支出 + 折旧摊销

行业调整：资本密集度不同导致EV/EBITDA不可直接跨行业比较
→ 必须以申万三级行业为分组计算分位数
```

#### P2方法：DCF（三阶段）

**适用场景**：消费龙头（茅台/五粮液）、公用事业（长江电力）、高速公路

```python

    """
    Stage 1 (years 1-5): 高增长 g1
    Stage 2 (years 6-10): 过渡增长 g2  
    Stage 3: 永续增长 terminal_g
    """
    # A股特殊：WACC计算需考虑
    # - 无风险利率（10年国债收益率）
    # - Beta（申万行业Beta均值 × 个股Beta调整）
```

**A股DCF最大风险**：盈利预测误差大 → 必须做敏感性分析（±20%EPS情景）

## 实施路线

### 阶段一：基础估值层（Week 1-3）

**目标**：跑通PE/PB分位数核心链路

1. **数据准备**
   - 从HDF5拉取4475只股的PE_TTM、PB_MRQ数据（按年度+季度）
   - 对齐年报/季报发布日期（使用**实际公告日**，避免使用假财报日）
   - 标记TTM数据（最近四个季度滚动）

2. **行业分位计算**
   - 对接PostgreSQL行业阈值库
   - 按申万三级计算每个行业的PE/PB分位数（当日快照）
   - 存入中间表：`valuation_snapshot(stock_code, trade_date, pe_ttm, pb_mrq, industry_pe_pct, industry_pb_pct)`

3. **格雷厄姆数基础版**
   ```
   GN = sqrt(22.5 × EPS_ttm × max(BVPS_ttm, 0.1))  -- BVPS防负
   margin_of_safety = (GN - price) / price × 100%
   ```

**交付物**：`valuation_engine_core.py` + 单元测试

### 阶段二：增强估值层（Week 4-6）

**目标**：EV/EBITDA + 调整后估值

1. **EV/EBITDA计算**
   - 需补充数据：带息负债（短期借款+长期借款+应付债券）、现金
   - 折旧摊销从HDF5已有字段获取

2. **行业调整系数库**
   - 基于历史回归，实证计算各申万三级行业的合理PE/PB倍数区间
   - 建立行业估值温度计（当前行业PE在历史区间的什么位置）

3. **格雷厄姆数行业调整版**
   - 按消费/金融/周期/科技/公用事业分类赋予调整系数

**交付物**：`valuation_enhanced.py` + 行业调整系数配置表

### 阶段三：高级估值层（Week 7-10）

**目标**：DCF + 估值信号综合输出

1. **DCF三阶段模型**
   - 盈利预测：使用近3年一致性预期 + 行业增速中位数
   - WACC：10年国债 + Beta×ERP，ERP用A股10年几何平均（~6.5%）
   - 敏感性矩阵：输出乐观/基准/悲观三档内在价值

2. **估值信号综合器**
   ```
   综合信号 = w1×PE分位信号 + w2×PB分位信号 + w3×EV_EBITDA信号 + w4×DCF信号 + w5×格雷厄姆信号
   权重建议：w1=0.25, w2=0.20, w3=0.20, w4=0.20, w5=0.15
   ```
   - 信号输出：低估/中性/高估/极端高估（破灭红旗触发时）

3. **红旗引擎联动**
   - 红旗触发 → 强制降权/标记不确定性
   - 例：红旗3（盈利可疑）触发时，DCF权重降至0，依赖PE/PB分位

**交付物**：`valuation_advanced.py` + `valuation_signal_aggregator.py`

| 市值 | market_cap / total_mv | ✅ 已有 |

### 需要补充的数据

| 数据项 | 优先级 | 来源 | 缺口评估 |
|--------|--------|------|---------|
| **PE_TTM**（实时） | P0 | 计算生成（股价/滚动净利润） | ⚠️ 需对齐TTM计算口径 |
| **PB_MRQ**（实时） | P0 | 计算生成（股价/最新净资产） | ⚠️ 需防净资产为负 |
| 带息负债（短期借款+长期借款+应付债券） | P1 | 财务报表，或聚源/东方财富 | 🔴 **缺口大**：需从年报提取，非标准字段 |
| 现金及等价物 | P1 | 财务报表 | 🔴 缺口：年报附注才有，季报通常无 |
| EV/EBITDA历史行业分位 | P1 | 计算生成 | ⚠️ 需历史PE/PB数据回溯 |
| 一致性预期EPS（未来2年） | P2 | 朝阳永续/同花顺iFinD | 🔴 **大缺口**：机构预测数据需采购 |
| 无风险利率（实时） | P1 | 10年国债收益率 | ✅ 可爬取或用tushare |
| Beta值（个股） | P2 | 申万/聚源 | 🔴 缺口：需计算或采购 |

### 数据缺口解决方案

```
优先级1（必须）：
- 带息负债：从年报PDF提取，或接入iFinD财务数据库
- 建议：先用"总负债 - 无息负债"粗估，后续精化

优先级2（重要）：
- 一致性预期：先用wind一致预期数据（如有），或用分析师覆盖股简化处理
- 个股Beta：使用申万三级行业Beta均值作为代理变量

优先级3（可选）：
- DCF敏感性分析的精确WACC：先用国债收益率 + 固定ERP，后续精化
```

## A股避坑指南

### 🔴 重大陷阱（必须规避）

#### 1. PE失真陷阱

**问题场景**：
- 周期股（钢铁/煤炭/化工）盈利高峰期，PE极低（5倍以下），但恰恰是高估信号
- 亏损股PE为负，PE排序完全失效
- 公司一次性处置资产导致EPS暴涨，PE失真

**解决方案**：
```
周期股：只用PB + EV/EBITDA，禁止使用PE
亏损股：PE标记为NaN，从排序中剔除
异常EPS：检测 (eps_q / eps_ttm_avg) > 3 → 标记为非经常性，切换至调整后EPS
```

#### 2. 银行股PE/PB陷阱

**问题场景**：
- 银行PE长期低于市场均值，PE分位法失效
- 银行PB<1不等于低估，可能是坏账担忧
- 地方银行与四大行业务风险截然不同

**解决方案**：
```
银行估值三合一：
① 调整PB = PB / (1 + (NPL_ratio - 行业均值NPL) × 5)
② ROA剔除拨备波动，看拨备前利润(PPOP)
③ 资负结构：关注存贷比、同业负债占比

个股与行业对比：不良生成率、拨备覆盖率、核心一级资本充足率
```

#### 3. 壳价值干扰

**问题场景**：
- A股特有的壳价值导致小市值股票PE/PB系统性偏高
- 亏损ST股PB<0.5可能反而是壳价值支撑，不代表低估
- 注册制推进后壳价值将加速消亡

**解决方案**：
```
市值分级估值：
① 总市值 < 30亿：壳价值修正，在PE/PB分位基础上额外下调5-10分
② 总市值 > 100亿：壳价值可忽略，正常估值
③ 注册制推进期（2024+）：逐步提高壳价值折价

动态跟踪IPO排队数量：排队家数>900时，壳价值溢价放大
```

#### 4. A股高波动性导致分位数不稳定

**问题场景**：
- A股熊市底部PE分位可能长期在10%以下（2013/2018/2022）
- 牛市顶部PE分位长期在90%以上（2007/2015）
- 历史10年窗口可能包含极端泡沫，污染分位数

**解决方案**：
```
双窗口分位：
- 5年窗口：捕捉近期估值趋势，更敏感
- 10年窗口：平滑极端值，更稳定

极端值重置机制：
if 当前PE > 历史PE_99pct: 截断至99分位
if 当前PE < 历史PE_1pct: 截断至1分位

情绪温度计：结合换手率、融资融券余额等情绪指标综合判断
```

#### 5. 新股/次新股估值失效

**问题场景**：
- 上市不满3年的新股，历史数据不足，分位数无法计算
- 科创板/创业板新股存在系统性高估值

**解决方案**：
```
新股处理规则：
① 上市 < 2年：标记为"数据不足"，分位计算返回NaN
② 使用行业均值替代历史分位
③ 科创板：行业PE上限锚定至80分位（抑制系统性高估干扰）

次新股：往前补齐上市前的财务数据（同口径模拟）
```

### 🟡 重要注意事项

#### 6. 盈利质量影响估值可信度

```
估值信号 + 红旗引擎联动：
- 红旗触发 → 降低估值置信度
- RF-001（大股东减持）：估值下调一档（低估→中性）
- RF-004（营收与现金流背离）：EV/EBITDA降权50%
- RF-006（商誉占净资产>50%）：PB下调20%
```

#### 7. 行业横向比较必须同一基准

```
错误做法：拿银行股PE和科技股PE直接比较
正确做法：
- 申万三级行业内比较
- 使用行业调整后PE（PE / 行业平均PE）作为标准化指标

行业调整PE = PE_ttm / 行业中位数PE
> 1.5 → 相对行业高估
< 0.7 → 相对行业低估
```

#### 8. 财报发布时滞

```
估值数据时效性管理：
- 年报发布截止：次年4月30日
- 季报发布截止：Q1为4月30日，Q2为8月31日，Q3为10月31日

使用规则：
① 年报发布前：使用上一年度全年数据（可接受）
② 年报发布后：立即更新为最新财务数据
③ 避免在财报空窗期（1-4月）使用过时财务数据做估值

建议：估值表增加"数据截止日"字段，辅助判断时效性
```

## 附加建议

### 与现有P0资产联动

```
估值引擎 × 红旗引擎 × 行业阈值库 联动架构：

[历史财务HDF5] ──┐
                ├──→ [估值引擎] ──→ [综合信号]
[行业阈值PG] ───┘         │
                           ↓
                   [红旗引擎联动]
                   - 红旗触发 → 调整估值权重
                   - 估值极端值 → 反向验证红旗

输出质量门控：
if 有效估值指标数 < 2 → 标记"数据不足，无法判断"
if 估值信号与红旗信号冲突 → 红旗优先，估值信号降权
```

### 性能优化建议

```
4475只股 × 10年历史 × 4个估值方法：
→ 约1.8M条估值记录/次全量计算

优化策略：
- 增量计算：非交易日仅更新当天有交易的股票
- 缓存：行业分位数按日更新，全市场股票共享
- 并行：按申万一级行业分组，16组并行计算
- 预计算：每日收盘后批量计算，存入Parquet，供实时查询
```
