## 修复1评价：置信度基于字符数判断

### 代码证据

`_validate_confidence`（locator.py L428-438）：
```
1500–15000字符 → 0.85
500–1500字符  → 0.60
其他          → 0.40
```

### 分析

**相比旧方法（keyword count）的优势**：

| 维度 | 旧方法（关键词命中数） | 新方法（字符数） |
|------|----------------------|-----------------|
| 度量对象 | 文本中是否出现预设词 | 内容是否充实完整 |
| 抗噪声能力 | 弱：关键词可能出现在非MD&A段 | 强：字符数直接反映内容量 |
| 与定位质量的关联 | 间接 | 直接 |
| 对截断的敏感性 | 低（截断文本仍含关键词） | 高（截断文本字符数明显偏少） |

字符数验证的是"**提取到了多少内容**"，而关键词命中验证的是"**内容是否相关**"。在置信度这个语境下，字符数更符合需求——它回答的是"这次定位+提取操作是否成功获取了足够的MD&A内容"。

**需要注意的问题**：
- **15000字符上限过于宽松**：A股年报MD&A章节一般在2000-8000字符，15000以上可能是合并了多个章节。超过15000字符给0.40反而惩罚了合理的丰富内容。建议改为20000或直接去掉上限（改为 `>50000 → 0.40` 才合理截断）。
- **阈值0.40偏低**：对于500字符以下（明显截断），0.40是合理的；但500-1500区间给0.60是恰当的，体现了"偏短但可用"的分级。

**结论**：✅ 方向正确，显著优于keyword count方法。建议后续调整上限阈值。

## 修复2评价：语义评分不再硬编码

### 代码证据

`_score_semantic`（scorer.py L169-227）：

```python
# 旧：直接 return 0.3
# 新：
base_from_ratio = 0.5 + ratio * 0.3   # 范围0.5-0.8
if valid_content:
    base_from_ratio = max(base_from_ratio, 0.6)
return max(score, base_from_ratio)
```

### 分析

**硬编码问题是否解决**：✅ 是的。

旧版本的问题是：只要`strategic_data`不为空，语义评分就与内容质量无关了——只和字段存在性有关（每个字段最多+0.3，总量可到1.0，但被下面的`base_from_ratio`下界约束）。

新版本引入了`ratio = filled/4`（填充字段比例），并用`0.5 + ratio*0.3`作为动态下界：
- 0个字段填充 → base=0.5
- 1个字段填充 → base=0.575
- 2个字段填充 → base=0.65
- 3个字段填充 → base=0.725
- 4个字段填充 → base=0.8

**关于测试结果 `semantic=0.50` 的解释**：

全部4年语义评分都是0.50，说明：
- `strategic_data` 不为 None（否则会返回0.3）
- 但`filled`接近0（否则base_from_ratio会>0.5）
- RuleBasedAnalyzer返回的结构是空的或只有部分字段有数据

这不是Fix 2的问题，而是**RuleBasedAnalyzer降级输出质量不足**的体现。从语义评分公式看：
```
return max(score, base_from_ratio)
```
只要`filled=0`（ratio=0），`base_from_ratio=0.5`，返回值的理论下限就是0.5。而实际测试返回0.50而非0.3，说明：
- `strategic_data` 是空dict `{}`（不是None）
- `filled = 0`，`ratio = 0`
- `score = 0.0`（没有字段触发加分）
- `valid_content = False`（没有有效内容）
- `base_from_ratio = 0.5 + 0*0.3 = 0.5`
- `return max(0.0, 0.5) = 0.5`

这是合理的——空数据结构返回0.5而非0.3，体现了"虽然没有有效内容，但系统尝试了提取"的中性判断。

**语义评分权重问题**：

语义评分在整体评分中仅占10%权重：
```
overall = 0.25*char + 0.30*structure + 0.35*keyword + 0.10*semantic
```
即使语义评分从0.3→0.5，对整体分数的贡献变化只有 `0.10*0.2 = 0.02`。因此语义评分对grade的影响有限，但这也是设计选择——当前权重下，关键词覆盖（35%）和结构（30%）是主导因素。

**结论**：✅ 正确解决了硬编码问题，语义评分现在会根据填充比例在0.5-0.8之间动态调整。

## 测试结果评估

```
2021: chars=2625 method=hierarchy confidence=0.85 semantic=0.50 grade=C
2022: chars=4888 method=hierarchy confidence=0.85 semantic=0.50 grade=B
2023: chars=5640 method=hierarchy confidence=0.85 semantic=0.50 grade=B
2024: chars=6311 method=hierarchy confidence=0.85 semantic=0.50 grade=B
```

- **4/5端到端成功率**：满意。2020年下载失败是外部问题，非代码缺陷。
- **全部使用层级推断（hierarchy）**：说明TOC书签定位在永新年报上未达到0.8门槛，层级推断在0.6-0.8之间成为最优方法。这是合理的降级行为。
- **confidence全部=0.85**：2625-6311字符全部落在1500-15000区间，置信度一致为0.85。这说明字符数验证的闸门效应是合理的——内容充实，置信度高。
- **semantic=0.50全部一致**：说明RuleBasedAnalyzer对所有年份都输出了类似的空/半空结构。这是后续需要优化的点，但不影响当前修复的合理性。
- **grade B/C是否可接受**：对于**管道原型阶段**，B/C完全可接受。原型阶段目标是验证管道能跑通、评分机制有效，A级需要LLM分析（非降级）和更精细的结构提取。

## 后续建议（不属于本次修复范围）

1. **调整`_validate_confidence`上限**：15000→20000或增加20000-50000区间的0.65档
2. **提升RuleBasedAnalyzer降级输出质量**：当前空结构导致semantic=0.50，可考虑在降级时给出一个"典型值baseline"而非空dict
3. **语义评分权重**：当前10%偏低，如果未来语义评分更准确，可以提升到15-20%
