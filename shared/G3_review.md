## 关键发现

### 1. 端到端100%是否可信

**结论：❌ 不可信 — 有严重代表性偏差**

| 问题 | 说明 |
|------|------|
| 样本同质性 | 5份年报全部来自**永新股份(002014)**，2020-2024连续5年；零跨公司验证 |
| 成功定义宽松 | `end_to_end_success=True` 只要5个stage无`Exception`抛出即成立，即使质量D级(0.55分)也算成功 |
| 跨公司风险未知 | 永新属于塑料软包装行业，格式规范；其他行业（如金融、地产、建筑）的PDF排版差异可能导致下载/定位失败率上升 |

**实际验证范围：** 仅同一公司5年，未测试不同公司、年报格式差异、行业多样性。

---

### 2. 层级推断置信度0.90 vs 预期65%

**结论：❌ 计算方式存在根本性误导**

**预期：** 层级推断(Level 2)预期准确率65%，整体定位率目标98%（4级降级叠加）。

**实际置信度计算（`_locate_hierarchy`）：**
```python
found_subs = sum(1 for v in sub_indices.values() if v)  # 统计4个分类中哪些有关键词命中
confidence = min(0.5 + found_subs * 0.1, 0.9)          # 4个全中 → 0.5+0.4=0.90
```

**问题：**
1. **置信度≠准确率**：0.90是"找到多少子节关键词"的计数，**不是**"章节边界是否正确"的准确率。两者完全不同。
2. **所有测试均为层级推断(0.90)**：5/5全部命中Level 2，未触发Level 1(TOC)或Level 3-4；这说明测试的年报恰好都有清晰的层级结构，不代表通用性。
3. **无交叉验证**：置信度0.90从未与人工标注或已知答案对比验证。
4. **代码注释明确说TOC(Level 1)覆盖率65%**，而测试全部绕过了Level 1，说明测试样本特意选择了有层级结构的年份，未覆盖TOC失效场景。

**改进建议：** 引入交叉验证机制，用已知MD&A段落的人工标注数据计算真实定位准确率，而非仅用关键词命中数代理置信度。

---

### 3. 规则引擎降级是否算成功

**结论：⚠️ 部分成功 — 存在隐性质量损失**

| 方面 | 评估 |
|------|------|
| 管道稳定性 | ✅ 规则引擎成功兜底，无`Exception`，管道完整运行 |
| 战略分析质量 | ⚠️ **严重损失**：语义得分=0.3（LLM正常0.7-1.0），质量被锁死在D-C级 |
| 幻觉控制 | ❌ **三层幻觉保障(Bounded LLM)全部失效**：规则引擎绕过Hard Constraints、Schema校验、一致性检查 |
| 可解释性 | ⚠️ 规则匹配标注为`[规则匹配]`，可溯源，但`source_quote`是静态文本片段，非原文精确引用 |

**关键问题：** 规则引擎降级后，战略分析数据的`hallucination_flags=[]`（无幻觉警告），给人"安全"的错觉，但实际上：
- 规则匹配精度低，可能混入无关上下文（`context[:80]`截断）
- 缺乏"原文未提及→NONE"的约束机制
- 无法区分"合理推断"和"规则误匹配"

**建议：** 规则引擎降级时，应明确标注`hallucination_flags=['rule_based_no_llm_constraints']`，并降级战略数据的置信度标签。

### 4. 质量评分D-C-B级是否可接受

**结论：⚠️ 勉强可接受 — 但存在结构性上限**

| 年份 | 等级 | 总分 | 字数分 | 结构分 | 关键词分 | 语义分 |
|------|------|------|--------|--------|----------|--------|
| 2020 | D | 0.55 | 0.30 (1950字) | - | - | 0.3 |
| 2021 | C | 0.61 | 0.50 (2625字) | - | - | 0.3 |
| 2022 | C | 0.70 | 0.75 (4888字) | - | - | 0.3 |
| 2023 | B | 0.77 | 0.75 (5640字) | - | - | 0.3 |
| 2024 | C | 0.74 | 0.75 (6311字) | - | - | 0.3 |

**分析：**
- 语义分=0.3是**硬编码兜底值**（`_score_semantic`中`if not strategic_data: return 0.3`），而非规则引擎的真实语义质量评估
- **即使规则引擎输出高质量内容，语义得分也不会超过0.3**，整体评分被锁死在C-D范围（除非字数分≥0.75）
- B级(2023)达到0.77，主要靠字数分拉高；语义质量贡献为零

**可接受条件：** 如果原型阶段目标是"管道能跑通"，D-C-B可接受；如果目标是"战略数据可用于投资分析"，则必须LLM正常工作才能达到A-B级。

## 问题列表

1. **[高优先级]** 置信度0.90计算方式错误——它是关键词命中计数，不是定位准确率；需引入人工标注的交叉验证
2. **[高优先级]** 测试样本严重同质化（永新1家公司5年），无法证明跨公司可扩展性；需补充至少5家不同行业公司的测试
3. **[高优先级]** 规则引擎降级时幻觉控制三层保障全部失效，但`hallucination_flags=[]`，应明确标注降级状态
4. **[中优先级]** 语义评分(0.3)是硬编码兜底值，未真实反映规则引擎输出质量；建议新增"规则引擎质量因子"
5. **[中优先级]** `end_to_end_success`定义过于宽松——质量D级(0.55分)也算成功；建议引入最低质量门槛（如≥C级或≥0.60分）
6. **[低优先级]** brotli问题(curl解决)虽已修复，但Level 2-4降级策略完全未在本次测试中触发，无降级链路的实际验证

## 总体评价

**有异议 — 原型管道技术可行，但验证严重不足，无法进入下一阶段。**

### 正面评价
- 管道架构合理，降级策略完整
- brotli问题修复（curl）有效
- 层级推断对结构化年报（永新）效果良好

### 否决理由
1. **置信度0.90是误导性指标**：它代表"4个子节关键词都命中"，不代表"章节边界正确"，与预期65%准确率无法对比
2. **零跨公司验证**：单一公司5年无法代表管道通用性
3. **LLM失效导致质量上限被锁死在C-D**：规则引擎降级虽"成功"，但战略数据质量不足以支撑投资分析场景
4. **幻觉控制机制在降级时完全失效**：这一点在测试中被掩盖（因为`hallucination_flags=[]`）

### G3Gate建议

| 建议 | 说明 |
|------|------|
| **不通过G3** | 验证充分性不足，需补充跨公司测试 |
| **条件通过（附加条件）** | 若下周补充5家不同行业公司×各1份测试，置信度重新计算，可考虑有条件通过 |
| **关键修复项** | 置信度计算方式、跨公司测试、幻觉控制降级标注 |
