## 一、监控指标定义

### 1.1 章节定位失败率 (Section Location Failure Rate)

| 指标 | 定义 | 计算公式 | 目标阈值 |
|------|------|----------|----------|
| 定位成功率 | MD&A章节正确定位的比率 | `成功数 / 总请求数` | ≥ 85% |
| 定位置信度 | 定位结果的可信度 | `confidence >= 0.6` 视为成功 | 均值 ≥ 0.7 |
| 方法降级率 | 降级到fallback的比率 | `fallback次数 / 总请求` | ≤ 15% |

**监控维度**：
- 按行业分组：食品饮料、金融、制造业等
- 按年度分组：2022/2023/2024
- 按定位方法分组：TOC书签、层级推断、关键词、fallback

### 1.2 规则引擎质量 (Rule Engine Quality)

| 指标 | 定义 | 计算公式 | 目标阈值 |
|------|------|----------|----------|
| 规则覆盖率 | 已实现规则数 / 应有规则数 | 需基线测量 | 动态基线 |
| 规则准确率 | 规则判定正确的比率 | `正确数 / 已触发数` | ≥ 90% |
| 红旗召回率 | 已知问题公司被正确识别的比率 | `召回数 / 已知问题公司` | ≥ 95% |
| 误报率 | 正常公司被误判的比率 | `误报数 / 正常公司` | ≤ 5% |
| 处理耗时 | 单公司规则引擎耗时 | 毫秒计 | ≤ 30s |

**监控维度**：
- 按行业分组：金融、消费、制造业等
- 按财务指标分组：ROE、毛利率、现金流等
- 按红旗类型分组：red/yellow/extreme

### 1.3 LLM幻觉率 (Hallucination Rate)

| 指标 | 定义 | 计算公式 | 目标阈值 |
|------|------|----------|----------|
| 结构化提取成功率 | LLM正确输出JSON的比率 | `有效JSON / 总请求` | ≥ 80% |
| 字段准确率 | 提取字段正确的比率 | `正确字段数 / 总字段数` | ≥ 85% |
| 事实一致性 | 与原文一致的比率 | 需人工抽检 | ≥ 90% |
| 幻觉触发率 | 检测到幻觉的比率 | `幻觉数 / 已分析数` | ≤ 10% |
| API错误率 | LLM API调用失败率 | `错误数 / 总请求` | ≤ 5% |

**监控维度**：
- 按模型分组：minimax/deepseek等
- 按文本长度分组：短/中/长
- 按行业分组：金融、消费等

## 二、告警阈值设计

### 2.1 阈值分级

| 级别 | 描述 | 触发条件 | 响应要求 |
|------|------|----------|----------|
| 🔴 P0 告警 | 严重阻塞 | 成功率 < 60% 或错误率 > 20% | 立即处理，1小时内响应 |
| 🟠 P1 告警 | 中等影响 | 成功率 60-75% 或错误率 10-20% | 4小时内响应 |
| 🟡 P2 告警 | 轻微偏差 | 成功率 75-85% 或错误率 5-10% | 24小时内响应 |
| ⚪ P3 提示 | 趋势预警 | 连续3天成功率下降 > 5% | 例行处理 |

### 2.2 维度特定阈值

**章节定位**：
```

    "P2": {"成功率<85%", "置信度均值<0.65"}
}
```


### 2.3 告警窗口

- **滑动窗口**：使用最近N条记录计算（N=100为默认值）
- **去噪**：排除明显异常值（如网络超时、API宕机）
- **冷却期**：同级别告警30分钟内不重复触发
