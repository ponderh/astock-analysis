## 1. 任务范围

### 1.1 模块定位

**投资结论引擎**是A股分析系统的最终输出层，负责：
1. **聚合**所有前置模块的分析结果
2. **综合评估**多维度投资因素
3. **输出**明确的投资建议和置信度
4. **生成结构化**的分析报告

### 1.2 输入数据源

| 模块 | 数据 | 用途 |
|------|------|------|
| 模块2 | 财务指标（ROE、营收增长、现金流等） | 基本面量化评估 |
| 模块5 | 红旗评分/警示信号 | 风险量化评估 |
| 模块6 | MD&A战略分析、风险因素 | 质地定性评估 |
| 模块7 | 公告情感分析 | 近期动态评估 |
| 模块9 | 公司治理评分 | 长期质地评估 |
| 行业阈值 | 行业基准数据 | 同业对比 |

### 1.3 输出成果

1. **投资建议**: 强烈买入 / 买入 / 持有 / 卖出 / 强烈卖出
2. **置信度**: 百分比（0-100%）
3. **评分雷达**: 各维度分数可视化
4. **理由摘要**: 核心逻辑100字
5. **风险提示**: 主要风险点列表
6. **分析报告**: 完整PDF报告

### 2.2 核心算法

#### 2.2.1 多因子评分模型

```python
# 输入因子及权重
WEIGHTS = {
    'financial_health': 0.30,      # 财务健康（模块2）
    'risk_score': 0.25,            # 风险评分（模块5）
    'quality_score': 0.20,         # 质地评分（模块6）
    'momentum_score': 0.15,        # 动量评分（模块7）
    'governance_score': 0.10,      # 治理评分（模块9）
}

# 最终得分计算
def calculate_final_score(factors: dict) -> tuple:
    """
    Returns: (recommendation, confidence, scores)
    """
    total = sum(factors[k] * WEIGHTS[k] for k in WEIGHTS)
    
    # 映射到投资建议
    if total >= 80:
        return ('强烈买入', total, factors)
    elif total >= 65:
        return ('买入', total, factors)
    elif total >= 45:
        return ('持有', total, factors)
    elif total >= 30:
        return ('卖出', total, factors)
    else:
        return ('强烈卖出', total, factors)
```

#### 2.2.2 数据缺失处理

| 缺失模块 | 处理策略 |
|----------|----------|
| 模块2财务 | 强制存疑，置信度上限50% |
| 模块5红旗 | 使用默认分（50分） |
| 模块6 MD&A | 跳过质地评分，使用均值替代 |
| 模块7公告 | 跳过动量评分，使用均值替代 |
| 模块9治理 | 使用均值替代 |

#### 2.2.3 阈值配置


## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 前置模块数据缺失 | 模块8无法运行 | 缺失容错机制，默认值 |
| 数据格式不一致 | 聚合失败 | 标准化转换层 |
| 评分逻辑争议 | 结论不准确 | 可调权重配置 |
| 循环依赖 | 与模块5/6互相调用 | 接口隔离 |
